Classes are designed to represent real world objects. So we could have a class called "Car" which has some attributes of "colour" and "registration code". It could have the methods of "accelerate" and "start engine".

Methods are the tasks that the object can do and the attributes are things which describe the object.

You will notice the use of the word "object" there. An "object" is what we call an "instance" of a class. For example, if we have a "Car" class then we would create objects of a car. These behave like variables, and we can save them into variables where we can perform tasks on the objects.

  • End Tutorial
  • Previous
  • Next

Let's have a look at an example:

<?php
class Car {
   //These are attributes
   private $colour;
   public $registration_no;

   //These are methods
   function __construct( $colour , $reg ) {
      //we use $this->... For class-level variables. Other variables are just local variables.
      $this->colour = $colour;
      $this->registration_no = $reg;
   }

   public function change_colour( $new_colour ) {
      $this->colour = $new_colour;
   }
}
?>

The final piece of code within the braces are the behaviours (methods or functions). Our sample behaviour is called "change_colour". You'll notice this is also "public". Methods can be public or private like the attributes.

So, to use the class, we would do the following:

<?php

The $my_car variable is now a Car object. We can access any of the public elements of the class using this variable.

For example, the attribute $registration_no is public, so we can "get" and "set" this like so:

<?php
//reading a value
echo "The registration number is " . $my_car->registration_no;
//writing a value
$my_car->registration_no = "JA93 PAU";
?>
